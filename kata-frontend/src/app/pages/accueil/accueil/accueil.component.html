Bank Account application base microservices

  • microservices based applications are attractive compared with monolithic ones
  • here are the microservices based application demo with business microservices apis:
    • api-customer: to manage customers
    • api-bank-account: to manage bank-accounts
    • api-operation: to manage operations carried out on bank-account
  • in this demo, each business microservice api has its own dependencies: db, configurations, etc
  • each business microservice api is implemented in architecture hexagonale
  • elements in each couples: (customer, bank-account), (bank-account, operation) communicates:
    • to manage a bank-account, api-bank-account gets customer from api-customer
    • to manage a operation, api-operation gets bank-account from api-bank-account
  • api-configuration-server: on start up each backend microservice api gets its configurations from a configuration server
  • api-gatway-service: we created and configured a gateway to route angular application requests
  • all backend microservices are deployed into docker images using docker engine
  • all docker containers of bancked microservices are orchestrated into a minikube cluster using Kubernetes
  • we created our fronted application with angular
